# CI/CD 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 DNF 增幅概率预测器建立 GitHub Actions CI + Vercel Serverless Function 部署的全自动化流水线。

**Architecture:** 后端 FastAPI 改为 Vercel Serverless Function，通过 `api/index.py` 入口导入 `backend/app/main.py` 的 app。前端保持 Vercel 静态站点不变。`vercel.json` 配置路由将 `/api/*` 转发到 serverless function，其余走前端 SPA。GitHub Actions 在 push/PR 时跑后端测试和前端构建验证。

**Tech Stack:** GitHub Actions, Vercel Serverless Functions (Python 3.11), pytest, Vite

---

## File Structure

| File | Action | Responsibility |
|------|--------|----------------|
| `api/index.py` | Create | Serverless Function 入口，导出 FastAPI app |
| `api/requirements.txt` | Create | Vercel 安装的后端依赖 |
| `vercel.json` | Modify | 增加 functions 和 rewrites 配置 |

| `.github/workflows/ci.yml` | Create | CI workflow：后端测试 + 前端构建 |

### Task 3: 修改 vercel.json

**Files:**
- Modify: `vercel.json`

- [ ] **Step 1: 更新 vercel.json**

将现有内容替换为：

```json
{
  "buildCommand": "cd frontend && npm run build",
  "outputDirectory": "frontend/dist",
  "installCommand": "cd frontend && npm install",
  "functions": {
    "api/index.py": {
      "runtime": "python-3.11"
    }
  },
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/api/index" },
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

变更说明：
- `functions`: 告诉 Vercel `api/index.py` 使用 Python 3.11 运行时
- `rewrites[0]`: 所有 `/api/*` 请求路由到 serverless function
- `rewrites[1]`: 其余请求走前端 SPA（保留原有行为）

### Task 4: 修改前端 API 基础地址

**Files:**
- Modify: `frontend/src/services/api.js:4`

- [ ] **Step 1: 修改 API_BASE 默认值**

将 `frontend/src/services/api.js` 第 4 行：

```javascript
const API_BASE = import.meta.env.VITE_API_BASE || 'http://localhost:8000'
```

改为：

```javascript
const API_BASE = import.meta.env.VITE_API_BASE || ''
```

原因：部署后前后端在同一域名下，`API_BASE` 为空字符串意味着请求会发到同域 `/api/*`，由 vercel.json 的 rewrites 路由到 serverless function。本地开发时设置 `VITE_API_BASE=http://localhost:8000` 即可。

- [ ] **Step 2: 验证前端构建**

### Task 5: 创建 GitHub Actions CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: 创建 workflow 文件**

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  backend-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: pip install -r backend/requirements.txt

      - name: Run tests
        run: pytest backend/tests/ -v

  frontend-build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: cd frontend && npm install

      - name: Build
        run: cd frontend && npm run build
```

说明：
- 两个 job 并行运行
- `backend-test`: Python 3.11 + pytest，跑 `backend/tests/` 下 5 个测试文件
- `frontend-build`: Node 20 + npm build，验证前端构建不出错
- 仅在 `main` 分支的 push 和 PR 触发

- [ ] **Step 2: 验证 YAML 语法**

Expected: `valid YAML`
